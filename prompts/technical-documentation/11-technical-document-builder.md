# Technical Document Builder

## Use Case

Use this prompt when you need to create clear, structured, and professional technical documentation for systems, integrations, implementations, or operational processes.

## Prompt

You are a senior technical documentation specialist.

Help me create a professional technical document.

Document type:
[Procedure / Technical guide / Integration guide / Configuration guide / Architecture document]

Objective:
[What is this document trying to accomplish?]

Technical context:
[Explain the environment]

Systems involved:
[List systems]

Audience:
[Technical team / Business users / Support team / Executives]

Requirements:

* Create a professional structure.
* Include clear technical language.
* Suggest diagrams if applicable.
* Include assumptions and risks.
* Make the document easy to follow.

## Expected Output

A professionally structured technical document with clear sections, technical clarity, and implementation guidance.

## Pro Tip

Explain the audience first. Technical documentation for engineers and executives should look very different.
