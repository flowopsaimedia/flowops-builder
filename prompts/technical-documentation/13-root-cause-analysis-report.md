# Root Cause Analysis Report

## Use Case

Use this prompt when incidents, failures, defects, or recurring issues require formal root cause analysis documentation.

## Prompt

You are a senior incident and root cause analyst.

Help me build a professional RCA report.

Incident summary:
[Describe the issue]

Business impact:
[Who was affected?]

Timeline:
[Sequence of events]

Symptoms observed:
[List symptoms]

Evidence:
[Logs / screenshots / observations]

Requirements:

* Identify probable root causes.
* Structure findings professionally.
* Suggest corrective actions.
* Include prevention recommendations.
* Prioritize clarity and evidence.

## Expected Output

A formal root cause analysis report with findings, recommendations, and preventive actions.

## Pro Tip

Provide timestamps and sequence of events to improve RCA quality.
