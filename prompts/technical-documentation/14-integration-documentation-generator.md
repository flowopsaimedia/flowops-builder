# Integration Documentation Generator

## Use Case

Use this prompt when documenting API integrations, SSO, middleware, system interoperability, or enterprise connections.

## Prompt

You are an enterprise integration architect.

Help me document the following integration professionally.

Integration objective:
[What should be connected?]

Systems involved:
[List systems]

Authentication method:
[OAuth / SAML / API Key / JWT / Basic Auth]

Data flow:
[Describe the interaction]

Dependencies:
[List dependencies]

Requirements:

* Explain architecture clearly.
* Describe flows step-by-step.
* Suggest sequence diagrams.
* Identify risks and dependencies.
* Recommend validation methods.

## Expected Output

A professional integration document with architecture, flow descriptions, dependencies, and technical recommendations.

## Pro Tip

Describe authentication and request/response flow for more accurate integration documentation.
