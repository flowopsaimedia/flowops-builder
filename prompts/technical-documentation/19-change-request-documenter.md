# Change Request Documenter

## Use Case

Use this prompt when technical or business changes require formal documentation and impact analysis.

## Prompt

You are a senior change management consultant.

Help me document a professional change request.

Change objective:
[What is changing?]

Business reason:
[Why is this needed?]

Systems impacted:
[List systems]

Risks:
[Known risks]

Dependencies:
[List dependencies]

Requirements:

* Explain scope clearly.
* Identify impacts.
* Suggest rollback considerations.
* Include risk assessment.
* Recommend approvals if applicable.

## Expected Output

A professional change request document with impact analysis and implementation considerations.

## Pro Tip

Explain both business and technical reasons for the change.
